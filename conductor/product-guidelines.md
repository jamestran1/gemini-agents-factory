# Product Guidelines: Gemini Agent Factory

## UX Philosophy: Simplicity First
- **Intuitive Design:** The interface should be minimalist and easy to navigate, ensuring that even non-technical founders can easily orchestrate their virtual teams.
- **Progressive Disclosure:** Expose essential features by default, tucking advanced configuration away to maintain an uncluttered workspace.
- **Clear Workflows:** The path from project inception to output delivery should be obvious and free of unnecessary friction.

## Prose Style: Professional & Direct
- **Clarity Over Cleverness:** Use clear, concise language. Avoid jargon where possible, and when necessary, ensure it is used correctly without fluff.
- **Action-Oriented:** Write instructions, buttons, and prompts starting with strong verbs (e.g., "Deploy," "Generate," "Review").
- **No Fluff:** Maintain a serious, corporate tone that instills confidence and trust.

## Visual Design: Modern & Clean
- **Whitespace & Typography:** Utilize ample whitespace to separate elements logically. Rely on highly legible, modern sans-serif fonts.
- **Color Palette:** Stick to a neutral, monochromatic base with a few intentional accent colors to highlight primary actions or statuses.
- **Consistency:** Ensure UI components (buttons, modals, forms) are uniform across all views to build an intuitive mental model.

## Error Handling: Informative & Guiding
- **Actionable Feedback:** When an agent encounters a problem or a user makes a mistake, the error message must explain both *what* went wrong and *how* to fix it.
- **Graceful Degradation:** The system should guide the user back to a functional state rather than leaving them at a dead end.
- **Transparency:** Agents should clearly log their assumptions and blockers so human orchestrators can quickly intervene and course-correct.
